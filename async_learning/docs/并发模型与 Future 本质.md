# 并发模型与 Future 本质

在深入任何异步编程代码之前，必须先想清楚一个问题：**我们为什么需要异步编程？**

---

## 1. 为什么需要异步？（剥离表象）

传统并发模型通常是 **1:1 物理线程模型** —— 每一个用户逻辑任务映射为一个操作系统内核线程。
对于 CPU 密集型任务，这种模型非常完美。但对于 **I/O 密集型任务**（例如网络服务器、等待数据库响应），内核线程会因为等待网络数据包或硬盘读写而被操作系统强行挂起：
- **高昂的栈内存**：每个内核线程默认分配 1MB~8MB 的虚拟栈内存。如果需要处理 10 万个并发连接，仅线程栈就会吃掉数百 GB 内存。
- **高昂的上下文切换成本**：线程在阻塞与就绪间的切换需要通过操作系统时钟中断实施抢占。这涉及从用户态陷入内核态，保存并恢复全部 CPU 寄存器，开销极其沉重。

为了打破线程开销的瓶颈，现代高并发语言都采取了**用户态协作式多任务**的设计思路，即用一段轻量级的数据结构（状态机）来代表任务，并在少量的 OS 线程上通过事件循环（如 epoll）进行非阻塞调度。

## 2. 五大主流并发模型深度对比

| 维度 | OS 线程 (1:1) | 事件驱动/回调 (Event-loop) | 绿色线程/强协程 (M:N, 如 Go) | Actor 模型 (Erlang) | 协作式异步 (Rust `async/await`) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **物理本质** | 操作系统时钟中断抢占调度 | 基于非阻塞事件驱动的回调执行 | 语言运行时调度器强介入的用户态协程 | 独立隔离的 Actor 间进行异步消息传递 | 编译器将异步块编译为**无堆分配的状态机** |
| **内存开销** | **大** (栈内存 1MB~8MB) | **极小** (仅闭包与状态变量) | **小** (Go 协程初始栈 2KB，可动态扩容) | **小** (极小的邮箱与独立堆栈) | **极极小** (编译期计算大小，**零堆分配**开销) |
| **上下文切换** | **极慢** (涉及陷入内核态、寄存器拷贝) | **极快** (仅普通函数指针跳转) | **快** (用户态调度，由 Go 运行时调度器完成) | **快** (由 Erlang VM 完成用户态调度) | **极快** (仅对状态机进行数值转移，等同于普通函数) |
| **心智负担** | **中** (死锁、锁竞争等共享内存问题) | **极高** (回调地狱，非线性控制流，错误极难传播) | **极低** (写起来和同步代码没有任何区别) | **中** (消息协议设计与状态隔离) | **高** (需处理借用检查、Pin 锁定与有色函数染色) |
| **运行时大小** | **无** (直接依赖 OS 机制) | **中** (包含事件循环与回调分流) | **庞大** (包含强 GC 垃圾回收与 M:N 调度器) | **庞大** (OTP 框架与 Erlang VM 运行时) | **无 / 极小** (标准库不提供运行时，由第三方库自由定制) |

## 3. Rust 异步的独特之处：“惰性拉模型”

大多数高级语言（如 Go、JavaScript、C#）的异步模型是 **“推（Push）模型”**：
- 当一个异步事件（如文件读取完毕）发生时，运行时（Runtime）会主动把对应的回调函数“推”入就绪队列等待执行。
- 这种模型开发心智负担低，但必须配备垃圾回收机制（GC）或庞大的动态内存分配，无法达到系统级语言对“零成本抽象”的苛求。

Rust 另辟蹊径，选择了 **“惰性拉（Poll-based）模型”**：
- `Future` 本质上是一个惰性状态机。当您调用 `let f = my_async_fn();` 时，**什么计算都不会发生**。
- Future 在未被 `poll` 之前，只是静静躺在内存中的数值，不占用任何额外的堆分配（可以在栈上），不消耗任何 CPU 循环。
- 必须由一个执行引擎（Executor）主动对其调用 `poll`，它才会睁开眼执行一步状态转移。
- 这种“拉”模型让 Rust 拥有了精细到寄存器级别的极致优化权，能做到零动态分配（Zero allocation），这使 Rust 异步能完美运行于嵌入式和操作系统内核等无堆环境中。

## 4. 有色函数（Colored Functions）染色体效应

异步编程带来高性能的同时，也引入了“染色体效应”：
- **同步函数**：普通的 `fn foo()`
- **异步函数**：`async fn bar()`
在 Rust 中，异步函数实际上是一个返回实现了 `Future` trait 匿名类型的特殊同步函数。这意味着你**不能在普通的同步函数中直接调用异步函数并期望得到其内部结果**。
* **原因**：因为 `bar()` 返回的是一个惰性 Future，必须在异步上下文中通过 `.await` 传递给 Executor 驱动它。
* **解法**：如果必须在同步代码中运行异步函数，必须通过 `block_on` 等阻塞工具强行启动一个局部事件循环，拉取该 Future 直至 Ready。
